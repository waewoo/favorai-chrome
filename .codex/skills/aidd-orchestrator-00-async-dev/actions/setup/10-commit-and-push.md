# 10 -- Commit and Push

Stages every file generated by previous actions, asks the user for explicit confirmation, then commits and pushes the branch.

## Input
- `answers` (required) -- config object from `02-ask-config`

## Output
```json
{
  "committed": true,
  "commit_sha": "abc1234",
  "branch": "main",
  "pushed": true
}
```


## Process

1. Skip this action when `answers.mode == "local"` and the user has no `.github/workflows/aidd-async.yml` to commit (i.e. only `local`-only artefacts exist; the local poll script and config can also be committed but it is opt-in for `local`-only users since their repo may not have CI at all).
2. Build the file list to stage based on `answers.mode`:
   - `remote`: `.github/workflows/aidd-async.yml`, `.claude/aidd-orchestrator.json`
   - `local`: `scripts/aidd-async-poll.sh`, `.claude/aidd-orchestrator.json`
   - `both`: union of the two
3. Show the user a `git status --short` of those paths. Ask "commit + push now? [y/N]".
4. On `y`:
   - `git add <listed paths>`
   - `git commit -m "ci: bootstrap aidd-orchestrator (mode=<answers.mode>)"`
   - `git push`
5. On `N`: print the exact `git add ... && git commit -m ... && git push` command for the user to run later. Exit cleanly without committing.
6. Emit the structured result.

## Test

After running and answering `y` on a clean branch: `git log -1 --pretty=%s` returns `ci: bootstrap aidd-orchestrator (mode=<mode>)`, `git diff origin/<branch>` is empty, and the listed paths are committed. After running and answering `N`: there is no new commit, no push, and stdout contains a copy-pasteable git command.
