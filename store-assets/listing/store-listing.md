## 2. LONG DESCRIPTION — ENGLISH

```
Are your bookmarks a graveyard of forgotten links, duplicates, and folders that made sense three years ago? FavorAI is the intelligent Chrome extension that fixes all of that — automatically — using the AI provider you already have.

With one click, FavorAI scans your entire bookmark library, detects problems, queries your AI, and proposes a clean, semantically organized structure. You review the proposed changes, select what to apply, and hit confirm. That's it.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
✨ CORE FEATURES
━━━━━━━━━━━━━━━━━━━━━━━━━━━

🔍 Smart Duplicate Detection
Automatically finds bookmarks pointing to the same URL across different folders. Keeps the best-placed copy and flags the rest for removal.

🔗 Dead Link Verification
Sends HEAD/GET requests to each bookmark URL and detects 404s, timeouts, and unreachable servers — so you can purge links that no longer exist.

🤖 AI-Powered Semantic Reorganization
Sends your bookmark titles and structure to your chosen AI provider, which analyzes the content and proposes a clean, logical folder hierarchy with intuitive names (emoji-prefixed, short enough to display on the bookmarks bar).

📋 Review Before You Commit
Every proposed change is listed in a checklist: folder creations, moves, renames, deletions. You stay in control — approve all, some, or none.

📜 Session History & Rollback
Every applied session is saved to your local history. Made a mistake? Roll back the entire session or individual changes with one click.

🌍 English & French Interface
The full UI adapts to your browser language automatically. (More languages planned.)

⚡ Two Reorganization Modes
• Minimal: only fixes clear semantic mismatches (e.g. "Docker Guide" sitting in "Personal"). Preserves your existing folder structure.
• Complete: full redesign — the AI rethinks your entire bookmark hierarchy from scratch with 6–8 top-level folders.

⚡ Compact Quick-Add Popup
Click the extension icon to bookmark the current tab instantly. The AI proposes the optimal folder and suggests a clean title. It also warns you of duplicates on the spot, with quick links to configure your API key or launch a full reorganization.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
⚙️ HOW IT WORKS
━━━━━━━━━━━━━━━━━━━━━━━━━━━

1. Configure your AI provider in the Configuration tab (OpenAI, Gemini, Claude, Mistral, DeepSeek, Grok, or a local Ollama instance). Enter your API key — it stays on your device.

2. Click "Minimal" or "Complete" to start analysis. FavorAI scans duplicates, checks dead links, then sends your bookmark structure to your chosen AI.

3. Review the proposed changes in the checklist. Each change shows exactly what will move where, what will be deleted, and what new folders will be created.

4. Click "Apply selected changes" to reorganize. FavorAI makes the changes directly in your Chrome bookmarks — no sync, no cloud.

5. Check the History tab to review what was applied, or click Rollback to undo any session instantly.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔌 SUPPORTED AI PROVIDERS
━━━━━━━━━━━━━━━━━━━━━━━━━━━

FavorAI works with any of the following providers (bring your own API key):

• 🟠 OpenAI — GPT-4o, GPT-4o mini, GPT-4 Turbo
• 🔵 Google Gemini — Gemini 1.5 Pro, Gemini 2.0 Flash
• ⚡ Anthropic Claude — Claude 3.5 Sonnet, Claude 3 Opus
• 🟣 Mistral AI — Mistral Medium, Mistral Large
• 🟢 DeepSeek — DeepSeek Chat, DeepSeek Reasoner
• 🤖 Grok (xAI) — Grok-3 and newer models
• 🏠 Ollama (Local) — Run any model 100% offline on your machine

You can also configure any custom OpenAI-compatible endpoint (vLLM, LM Studio, etc.).

━━━━━━━━━━━━━━━━━━━━━━━━━━━
🔒 PRIVACY & SECURITY
━━━━━━━━━━━━━━━━━━━━━━━━━━━

FavorAI is built with a strict privacy-first architecture:

• No FavorAI servers. There is no backend, no database, no user accounts. Nothing leaves your browser except what goes to the AI provider you explicitly configured.

• Your API key is stored locally in chrome.storage.sync, encrypted by Chrome, and never transmitted to FavorAI.

• Bookmark data is sent only to your AI provider. Titles, folder structure, and URLs are included in the payload to help with semantic classification. You choose the provider — you control where the data goes.

• No telemetry, no analytics, no tracking pixels of any kind.

• All session history is stored locally in chrome.storage.local and can be cleared at any time from the History tab.

• Uninstalling the extension permanently removes all stored data.

━━━━━━━━━━━━━━━━━━━━━━━━━━━
💡 TIPS FOR BEST RESULTS
━━━━━━━━━━━━━━━━━━━━━━━━━━━

• Start with Minimal mode to get a feel for the AI's suggestions before committing to a Complete reorganization.
• For large libraries (200+ bookmarks), use the folder selector to analyze one section at a time.
• GPT-4o and Claude 3.5 Sonnet tend to give the most accurate semantic reorganizations.
• Ollama with Llama 3 works well for users who prefer full privacy (no data leaves your machine).
• The "Check dead links" option slows analysis but is worth running every few months on large libraries.
```

## 6. JUSTIFICATION FOR `<all_urls>` PERMISSION

> Required by the Chrome Web Store review team when using `optional_host_permissions: ["<all_urls>"]`

```
FavorAI requests the <all_urls> optional host permission for two specific purposes:

1. Dead link verification: When the user enables "Check dead links", the extension
   sends HEAD/GET requests to each bookmark URL to detect 404s and unreachable servers.
   Since bookmarks can point to any domain, access to all URLs is required.

2. Custom LLM endpoint support: Users can configure any OpenAI-compatible endpoint
   (including local servers like Ollama at http://localhost:11434 or custom vLLM
   deployments). The extension cannot predict the domain in advance.

This permission is OPTIONAL and is only requested at runtime when the user
explicitly enables the dead link checking feature via the checkbox in the UI.
All other features (duplicate detection, AI reorganization via standard providers,
session history) work without this permission.
```
